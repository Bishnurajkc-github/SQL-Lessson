# AI_CONTEXT.md – SQL Learning (Parent & Daughter)

## Purpose

This project is for learning **SQL using SQLite from the absolute beginning**.

There are two learners:

* **Parent:** learning SQL first to build a strong foundation for future **Python + SQLite + Flask** projects, and to confidently teach the daughter.
* **Daughter:** studying SQLite as part of **Grade 10 computer science**, where school teaching is more theory-based, so this project adds practical understanding.

---

## Main Learning Goals

* Understand SQL concepts instead of memorizing syntax.
* Learn by creating tables, inserting data, querying data, and later modifying it safely.
* Build confidence through repeated hands-on practice.
* Progress from beginner to intermediate SQL in a structured order.
* Connect SQLite learning to future Flask, CRUD, and database work.

---

## Practice Environment

### Current setup

* **VS Code**
* **SQLite terminal / shell**
* lesson files such as `lesson01.sql`, `lesson02.sql`, etc.
* persistent database file: `school.db`

### Standard workflow

1. Open **PowerShell** in the project folder.
2. Run `sqlite3`
3. Open the real database with `.open school.db`
4. Run lesson files with `.read lesson01.sql`, `.read lesson02.sql`, etc.
5. Verify with `.databases`, `.tables`, `.schema students`, and `SELECT * FROM students;`

---

## What we did

## Main concepts learned

### Lesson 16 – `UPDATE`

Learned how to change existing values in rows using:

```sql
UPDATE students
SET city = 'Lalitpur'
WHERE id = 3;
```

Key ideas:

* `SET` decides which column value changes
* `WHERE` decides which row(s) change
* `UPDATE` without `WHERE` can change many or all rows
* `UPDATE` can also be used to fill or restore `NULL` values

### Lesson 17 – `DELETE`

Learned how to remove row(s) from a table using:

```sql
DELETE FROM students
WHERE id = 8;
```

Key ideas:

* `DELETE` removes full rows, not just one column
* `WHERE` decides which row(s) are removed
* `DELETE` without `WHERE` can remove all rows in the table

---

## Important practice incident and correction

During Lesson 16 practice, the row with `id = 11`—which had been intentionally kept as a `NULL` practice row—was accidentally changed so that:

* `city` became `Lalitpur`
* `marks` became `100`

That row was then corrected back to its intended state with:

```sql
UPDATE students
SET city = NULL,
    marks = NULL
WHERE id = 11;
```

This was a useful reminder that practice rows should be protected when they are being kept for later lessons.

---

## Important safety habit reinforced

For both `UPDATE` and `DELETE`, the safe workflow is:

1. `SELECT` first to see which row(s) match
2. run `UPDATE` or `DELETE`
3. `SELECT` again to verify the result

That habit matters because Phase 3 commands can change or remove data permanently from the practice table.

# 2) `share_docs.ps1`

## Purpose

This script collects all `.md` files inside the `docs/` folder and combines them into one file:

```text
all_docs_for_chatgpt.txt
```

It then opens that file in Notepad so I can copy everything and paste it into ChatGPT.

This is useful when I want to share:

* project context
* progress notes
* learning log
* roadmap
* history
* questions and reminders

all at once.

---

## Where the script should be stored

Store it in the **project root**, not inside `docs/`.

Correct example:

```text
my_flask_project/
├─ share_docs.ps1
├─ docs/
└─ ...
```

---

## Important distinction: file vs command

### Script file

This is the actual file saved in the project root:

```text
share_docs.ps1
```

### Command used to run it

This is typed in PowerShell:

```powershell
.\share_docs.ps1
```

The command is **not a file**. It simply tells PowerShell to run the script from the current folder.

---

## What `.\share_docs.ps1` means

### `.\`

Means:

> “look in the current folder”

### `share_docs.ps1`

Means:

> “run the PowerShell script file with this name”

So:

```powershell
.\share_docs.ps1
```

means:

> “Run the file named `share_docs.ps1` from the current folder.”

---

## How to run it

### Step 1: open PowerShell in the project folder

Example:

```powershell
cd C:\Users\Bishnu\Desktop\my_flask_project
```

### Step 2: run the script

```powershell
.\share_docs.ps1
```

---

## What the script does behind the scenes

When run, it:

1. checks the `docs/` folder
2. finds all `.md` files
3. reads the content of each file
4. combines them into:

   ```text
   all_docs_for_chatgpt.txt
   ```
5. opens that file in Notepad

This makes it easy to share all project notes in one go.

---

# 3) Full code for `share_docs.ps1`

Keep this code inside the file named `share_docs.ps1` in the project root:

```powershell
# share_docs.ps1
# Run this from your project root.
# It combines all .md files from the docs folder into one text file for easy sharing with ChatGPT.

$docsPath = ".\docs"
$outputFile = ".\all_docs_for_chatgpt.txt"

if (-not (Test-Path $docsPath)) {
    Write-Host "docs folder not found at $docsPath"
    exit
}

# Clear old export file if it exists
Remove-Item $outputFile -ErrorAction SilentlyContinue

Get-ChildItem "$docsPath\*.md" | Sort-Object Name | ForEach-Object {
    Add-Content $outputFile "`n`n=================================================="
    Add-Content $outputFile "FILE: $($_.Name)"
    Add-Content $outputFile "=================================================="
    Get-Content $_.FullName | Add-Content $outputFile
}

Write-Host ""
Write-Host "Done. Created: $outputFile"
Write-Host "Opening file..."
notepad $outputFile
```
